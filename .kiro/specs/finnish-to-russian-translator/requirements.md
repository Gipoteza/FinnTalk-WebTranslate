# Документ требований

## Введение

Браузерное расширение для автоматического перевода финских веб-страниц на русский язык с использованием API ChatGPT. Расширение автоматически определяет тематику страницы, выбирает соответствующий системный промпт для перевода, поддерживает несколько стилей перевода и опциональную проверку качества. Пользователь управляет расширением через минималистичный Popup UI, а настройки хранятся на странице Options.

## Глоссарий

- **Расширение** — браузерное расширение (Chrome/Firefox) для перевода финских страниц на русский язык.
- **Content_Script** — скрипт, внедряемый в страницу; извлекает текст и заменяет его переводом без нарушения HTML-структуры.
- **Background_Script** — фоновый сервис-воркер; управляет очередью API-запросов, повторными попытками и обработкой ошибок.
- **Popup_UI** — всплывающая панель расширения с кнопками управления переводом.
- **Options_Page** — страница настроек расширения.
- **API_Client** — модуль Background_Script, отвечающий за взаимодействие с OpenAI ChatGPT API.
- **Тематика** — категория содержимого страницы: медицинская, юридическая, финансовая, техническая, IT, бытовая, деловая, разговорная.
- **Стиль перевода** — режим перевода, выбираемый пользователем: буквальный, литературный, нейтральный.
- **Блок текста** — фрагмент текста страницы размером от 2000 до 4000 символов, передаваемый в одном API-запросе.
- **Системный промпт** — инструкция для ChatGPT, формируемая на основе определённой тематики и выбранного стиля перевода.
- **Проверка качества** — опциональный третий API-запрос для стилистической корректуры перевода.
- **API-ключ** — секретный ключ доступа к OpenAI API, хранимый в защищённом хранилище.

## Требования

### Требование 1: Определение тематики страницы

**User Story:** Как пользователь, я хочу, чтобы расширение автоматически определяло тематику финской страницы, чтобы перевод учитывал специфику предметной области.

#### Критерии приёмки

1. WHEN пользователь инициирует перевод страницы, THE Content_Script SHALL извлечь текстовое содержимое страницы и передать его в Background_Script.
2. WHEN Background_Script получает текст страницы, THE API_Client SHALL отправить первый запрос к ChatGPT API с промптом: "Определи основную тематику текста (медицина, юриспруденция, финансы, техника, IT, бытовая, деловая, разговорная). Верни только одно слово."
3. WHEN ChatGPT API возвращает ответ на запрос определения тематики, THE Background_Script SHALL распознать одну из восьми тематик: медицина, юриспруденция, финансы, техника, IT, бытовая, деловая, разговорная.
4. IF ответ ChatGPT API не соответствует ни одной из восьми тематик, THEN THE Background_Script SHALL присвоить тематику "бытовая" по умолчанию.
5. WHEN тематика определена, THE Background_Script SHALL сформировать Системный_промпт, соответствующий определённой тематике и выбранному стилю перевода.

---

### Требование 2: Перевод текста страницы

**User Story:** Как пользователь, я хочу получить качественный перевод финской страницы на русский язык, чтобы понимать её содержимое.

#### Критерии приёмки

1. WHEN тематика страницы определена, THE Content_Script SHALL разбить текст страницы на Блоки_текста размером от 2000 до 4000 символов с сохранением границ предложений.
2. WHEN Блок_текста сформирован, THE API_Client SHALL отправить второй запрос к ChatGPT API с Системным_промптом и текстом блока для перевода с финского на русский язык.
3. WHEN API_Client получает переведённый Блок_текста, THE Content_Script SHALL заменить оригинальный текст в DOM переведённым текстом без изменения HTML-структуры страницы.
4. THE Content_Script SHALL обрабатывать Блоки_текста последовательно, обновляя страницу по мере получения переводов.
5. WHILE перевод страницы выполняется, THE Popup_UI SHALL отображать индикатор прогресса с указанием количества обработанных блоков.

---

### Требование 3: Стили перевода

**User Story:** Как пользователь, я хочу выбирать стиль перевода, чтобы результат соответствовал моим предпочтениям.

#### Критерии приёмки

1. THE Popup_UI SHALL предоставлять пользователю выбор из трёх стилей перевода: буквальный, литературный, нейтральный.
2. WHEN пользователь выбирает стиль перевода, THE Background_Script SHALL использовать выбранный стиль при формировании Системного_промпта для всех последующих запросов перевода.
3. THE Background_Script SHALL применять стиль "нейтральный" по умолчанию при первом запуске расширения.
4. WHEN пользователь изменяет стиль перевода во время активного перевода, THE Background_Script SHALL применить новый стиль начиная со следующего необработанного Блока_текста.

---

### Требование 4: Проверка качества перевода

**User Story:** Как пользователь, я хочу получать стилистически корректный перевод, чтобы текст звучал естественно на русском языке.

#### Критерии приёмки

1. WHERE проверка качества включена в настройках, THE API_Client SHALL отправить третий запрос к ChatGPT API для сравнения оригинального и переведённого текста и исправления стилистических ошибок.
2. WHERE проверка качества включена, WHEN API_Client получает скорректированный текст, THE Content_Script SHALL заменить предварительный перевод скорректированным текстом на странице.
3. WHERE проверка качества включена, THE Background_Script SHALL выполнять проверку для каждого Блока_текста после получения его перевода.
4. THE Options_Page SHALL предоставлять пользователю переключатель для включения и отключения проверки качества.

---

### Требование 5: Управление через Popup UI

**User Story:** Как пользователь, я хочу управлять переводом через удобный интерфейс, чтобы быстро запускать и контролировать перевод.

#### Критерии приёмки

1. THE Popup_UI SHALL содержать кнопку "Перевести страницу" для запуска однократного перевода текущей страницы.
2. THE Popup_UI SHALL содержать кнопку "Автоперевод" для включения автоматического перевода при загрузке каждой новой страницы.
3. THE Popup_UI SHALL содержать кнопку "Показать оригинал" для переключения между переведённым и оригинальным текстом страницы.
4. WHEN пользователь нажимает "Показать оригинал", THE Content_Script SHALL восстановить оригинальный текст страницы, сохранённый до начала перевода.
5. WHEN пользователь повторно нажимает "Показать оригинал" после восстановления оригинала, THE Content_Script SHALL отобразить ранее полученный перевод без повторного обращения к API.
6. THE Popup_UI SHALL отображать текущую определённую тематику страницы после завершения первого API-запроса.

---

### Требование 6: Настройки расширения

**User Story:** Как пользователь, я хочу настраивать параметры расширения, чтобы контролировать использование API и качество перевода.

#### Критерии приёмки

1. THE Options_Page SHALL предоставлять поле для ввода и сохранения API-ключа OpenAI.
2. THE Options_Page SHALL предоставлять выбор модели ChatGPT из списка доступных моделей (например, gpt-4o, gpt-4o-mini, gpt-3.5-turbo).
3. THE Options_Page SHALL предоставлять настройку лимита запросов в минуту для предотвращения превышения квоты API.
4. WHEN пользователь сохраняет API-ключ, THE Options_Page SHALL сохранить его в защищённом хранилище браузера (chrome.storage.local или аналог).
5. THE Расширение SHALL передавать API-ключ исключительно в запросах к домену api.openai.com.

---

### Требование 7: Обработка ошибок и надёжность

**User Story:** Как пользователь, я хочу получать понятные уведомления об ошибках и автоматическое восстановление при сбоях, чтобы перевод работал надёжно.

#### Критерии приёмки

1. IF запрос к ChatGPT API завершается ошибкой сети, THEN THE Background_Script SHALL повторить запрос до 3 раз с экспоненциальной задержкой (1с, 2с, 4с).
2. IF все повторные попытки запроса исчерпаны, THEN THE Background_Script SHALL уведомить пользователя об ошибке через Popup_UI с описанием причины сбоя.
3. IF ChatGPT API возвращает код ошибки 401, THEN THE Background_Script SHALL уведомить пользователя о неверном или отсутствующем API-ключе и открыть Options_Page.
4. IF ChatGPT API возвращает код ошибки 429, THEN THE Background_Script SHALL приостановить очередь запросов на 60 секунд и уведомить пользователя о превышении лимита.
5. THE Background_Script SHALL записывать все ошибки API в журнал (console.error) с указанием кода ошибки, временной метки и идентификатора запроса.
6. IF пользователь пытается запустить перевод без сохранённого API-ключа, THEN THE Popup_UI SHALL отобразить сообщение с предложением перейти на Options_Page для ввода ключа.

---

### Требование 8: Сохранение HTML-структуры

**User Story:** Как пользователь, я хочу, чтобы после перевода страница выглядела корректно, чтобы вёрстка и навигация не нарушались.

#### Критерии приёмки

1. THE Content_Script SHALL извлекать только текстовые узлы DOM, не затрагивая HTML-атрибуты, теги скриптов и теги стилей.
2. WHEN Content_Script заменяет текст в DOM, THE Content_Script SHALL изменять только содержимое текстовых узлов, сохраняя все HTML-теги и атрибуты неизменными.
3. THE Content_Script SHALL пропускать текстовые узлы, содержащие менее 3 символов, для исключения служебных символов и пробелов из обработки.
4. THE Content_Script SHALL сохранять оригинальные текстовые узлы в памяти до завершения перевода для обеспечения функции "Показать оригинал".
